# 后端

后端需要负责房间的管理

游戏定位：局域网内的游戏房间，允许玩家中途加入

1.登录部分：不需要使用密码，用户自己输入一个用户名即可进入游戏，但是由后端生成一个UUID来唯一标识一个用户。

游戏区域是12150单位

2.初始化随机生成各个玩家的初始位置（各个玩家之间的初始距离不能小于4050个单位），初始航向；初始位置随机，初始航向随机，初始潜深0m，初始航速为0

以及初始化生成目标船队的初始位置，目标船队22艘船。每条船的航速相同，航向相同。

商船的航行路线固定直线，但是起点和终点由后端随机确定。目标船队的航行路线从地图的一个边缘移动到另外一个边缘。

商船队的队形结构大体如下

```
商船  商船  商船  商船  商船

商船  商船  商船  商船  商船

商船  商船  商船  商船  商船

商船  商船  商船  商船  商船
---------------------------------

横向列间距：约 1000 yards ≈ 914 m（历史数据，不是项目需要的数据）
横向间距261个单位

同一列前后船距：约 500–800 yards ≈ 457–732 m
纵向间距130到210个单位（建议160个单位）

商船之间不需要严格对齐。要看起来乱一点，即每艘船大概偏离其标准位置大约0-50个单位（横向和纵向都要有一定的偏移量）
```

生成这些信息之后立刻返回给客户端。

3.后端负责维护目标船队，每条船的航向，航速，位置，是否被击沉等信息。并将目标船队的最新消息传送给各个客户端。

大约150～200ms一次

4.客户端将各自的潜艇消息传送给后端；航向，航速，潜深，鱼雷数量。后端收集数据后，将数据汇总并发送给各个客户端

大约100ms一次

5.接收客户端发来的击沉记录，更新击沉记录，更新目标船队信息。并立刻将新的船队信息反馈给客户端。

6.客户端（潜艇）之间应该能互相沟通信息，后端接收客户端的普通消息，并且转发，推送给对应的客户端（单播或广播）
